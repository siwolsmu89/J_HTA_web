HTML
	- HyperText Markup Language
		* 웹에서 사용되는 문서를 작성할 때 사용되는 마크업(태그)를 정의해 놓은 언어
		* HyperText : 문서의 링크를 클릭하면 연결된 문서로 즉시 이동할 수 있는 문서
		* Markup : 문서를 구성하고 있는 여러 요소를 정의할 때 사용되는 태그(표시자)
			   (문서의 구성요소 : 제목, 내용, 리스트, 표, 그림, 링크 등)

	-  HTML의 구성요소
		<!DOCTYPE html>	<!-- HTML 문서의 유형(형식)을 선언하는 선언문이다. -->
		<html> 		<!-- 이 문서가 HTML 문서임을 나타낸다. -->
			<head>
				<!-- 웹 브라우저가 HTML 문서를 해석할 때 필요한 정보를 정의한다.-->
			</head>
			<body>
				<!-- 웹 브라우저에 실제로 표시되는 컨텐츠를 정의한다. -->
			</body>
		</html>
		* HTML 문서는 <html> 태그를 하나만 포함할 수 있다.
		* <html> 태그는 <head> 태그와 <body> 태그를 각각 하나씩 포함할 수 있다.
		* <html> 태그는 이 문서가 html 문서임을 정의한다.
		* <head> 태그는 웹 브라우저가 이 문서를 해석하는 데 필요한 정보를 제공한다.
		* <head> 태그에 정의된 내용은 화면에 표시되지 않는다.
		* <body> 태그는 웹 브라우저에 실제로 표시될 내용을 포함한다.
		* <body> 태그는 문서의 제목, 내용, 그림, 동영상, 링크, 표 등을 포함한다.

	- HTML 문서의 구성요소
		* 태그
			- 태그는 컨텐츠를 담는 그릇이다.
			- 태그는 태그마다 그 용도가 미리 정해져 있다.
			- 태그의 이름은 소문자로 적는다.
			- 태그에는 여는 태그와 닫는 태그가 있다.
				* 여는 태그 : <태그명>
					      <h1>, <p>, <a> 등
				* 닫는 태그 : </태그명>
					      </h1>, </p>, </a> 등
				* 잘못된 태그 작성 : <H1>, < h1>, < /h1>, </ h1> 등
			- 태그를 작성할 때는 여는 태그와 닫는 태그의 짝을 맞추어야 한다.
				* 올바른 예
					<h1>주요 뉴스</h1>
					<p>오늘의 뉴스입니다.</p>
				* 잘못된 예
					<h1>주요 뉴스
					<p>오늘의 뉴스입니다.</h1></p>
			- 컨텐츠는 여는 태그와 닫는 태그 사이에 적는다.
			- 컨텐츠는 텍스트나 다른 태그가 될 수 있다.
				* 컨텐츠가 text인 경우
					<p>텍스트 컨텐츠입니다.</p>
				*컨텐츠가 다른 태그인 경우
					<div><h1>컨텐츠입니다.</h1></div>
			- 태그들 사이에는 부모태그, 자식태그의 관계를 갖는 태그가 있다.
				* 자식태그는 부모태그 안에서만 사용할 수 있다.
				* 자식태그를 부모태그 안에 적을 때는 들여쓰기를 한다.
				* 예)
					<head>의 부모는 <html>이다.
					<body>의 부모는 <html>이다.
					<li>의 부모는 <ul> 혹은 <ol>이다.
					<thead>, <tbody>, <tfoot>의 부모는 <table>이다.
				* 올바른 예)
					<ul>
						<li>커피</li>
						<li>주스</li>
					</ul>
				* 잘못된 예)
					<p>
						<li>커피</li>
						<li>주스</li>
					</p>
			- 태그 중에는 컨텐츠를 가지지 않는 태그도 있다.
				* 여는태그 닫는태그의 형태로 적지 않는다.
				* 예)
					<img />, <hr />, <input />
			- 태그는 하나 이상의 속성을 가질 수 있다.
				* 속성은 그 태그와 관련된 부가적인 정보, 추가기능을 정의할 때 사용한다.
				* 모든 태그가 공통으로 가지고 있는 속성도 있고, 특정 태그만 가지고 있는 속성도 있다.
				* 속성은 이름과 값으로 구성되어 있다.
					<태그 속성명="속성값">
				* 하나의 태그에 동일한 속성명을 여러 번 적을 수 없다.
				* 태그에 여러 개의 속성을 적을 때 특별한 순서는 없다.
					<img src="logo.png" width="50" height="50" />
					<img width="50" height="50" src="logo.png" />
					<img height="50" src="logo.png" width="50" />
					* 모두 유효한 작성법이다.
				* 속성은 본래의 사용목적에 맞게 작성해야 한다.
				* 임의의 속성을 마음대로 추가할 수 없다.
				* 어떤 속성은 속성값도 미리 정해진 값들 중에서만 사용해야 한다.
					<input type="속성값">
						유효한 속성값 : text, password, checkbox, radio, file, hidden, ...
				* data-*로 시작하는 속성은 임의로 추가할 수 있다. (브라우저가 무시함)
					<img src="김수영.png" data-job="가수"
							      data-genre="인디"
							      data-company="매직스트로베리사운드" />
		* 속성
		* 컨텐츠
